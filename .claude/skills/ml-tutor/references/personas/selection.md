# 导师性格选择

本文件定义性格的**动态发现**与**选择写入**流程。
触发时机：profile 文件中「已选风格」字段为空时。

---

## 动态发现流程

**本文件不维护任何固定的性格列表。**
选项完全由 Skill目录下 `references/personas/` 中的文件动态生成。

执行步骤：

1. 从 profile 的「基础配置」读取 `Skill目录` 的绝对路径
2. 列出 `<Skill目录>/references/personas/` 下所有 `.md` 文件
3. 排除 `selection.md` 自身
4. 逐个读取每个文件**开头的 YAML frontmatter**，提取 `name` 和 `description` 字段
5. 跳过缺少这两个字段的文件（不报错，静默忽略）
6. 将有效结果按**文件名字母顺序**排列，构建编号选项列表

---

## 向用户展示

用以下格式呈现（`name` 和 `description` 来自动态发现的结果）：

> 我可以用不同的风格来陪你学，你喜欢哪种？
>
> **① [name]** — [description]
> **② [name]** — [description]
> **③ [name]** — [description]
> ...（有几个显示几个）

---

## 接收用户回答

- 用户输入**序号**（如"2"、"②"）→ 对应序号的文件
- 用户输入**关键词**（如"萌妹"、"教授"）→ 模糊匹配 `name` 字段，命中则确认
- 用户输入**模糊**（如"随便"、"你选"）→ 默认第一个选项，告知用户
- 匹配到多个候选 → 列出候选让用户二次确认

---

## 写入 Profile

确认选择后，写入 profile 的「导师性格」区块：

```
已选风格：<文件名，如 cute-girl.md>
性格文件：<Skill目录绝对路径>/references/personas/<文件名>
```

> 存储**文件名**和**性格文件的绝对路径**。绝对路径确保后续任何工作目录下都能直接读取，不依赖当前 pwd。

写入完成后，立即读取对应的 persona 文件（使用绝对路径），后续对话全程按该性格响应。

---

## 扩展约定

新增一个性格，只需在 `references/personas/` 目录下放一个 `.md` 文件，
frontmatter 包含 `name` 和 `description` 字段即可。
**无需修改本文件或 SKILL.md 的任何内容。**

```yaml
---
name: 你的性格名称
description: 一句话描述这个性格的风格特点
---
```
