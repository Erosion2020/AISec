# 线性代数：空间变换的语言

线性代数不是用来解方程的，它是用来**描述空间里的变换**的。
矩阵不是数字表格，它是一个**动作**——对空间的拉伸、旋转、翻转。
带着这个视角，线代里所有东西都会变得有直觉可循。

---

## 向量：空间里的箭头

**直觉**：向量是空间里一根有方向、有长度的箭头，不是一列数字。
数字只是告诉你箭头的终点在哪里（从原点出发）。

**例子**：向量 [3, 2] 的意思是：往右走 3 步，往上走 2 步，就是这根箭头的终点。

**ML 里的用法**：
- 一张图片可以表示成一个向量（每个像素值是一个分量）
- 神经网络每层的输入输出都是向量
- 注意力机制里的 Query、Key、Value 都是向量

---

## 矩阵：对空间的变换操作

**直觉**：矩阵是一个**操作**，它把空间里所有的箭头都按同一个规则变换。
把矩阵乘以一个向量，就是在对那根箭头做变换。

**例子**（2×2 矩阵，手算可完成）：

矩阵 M = [[2, 0], [0, 3]]，向量 v = [1, 1]

M 乘以 v 的计算：
```
结果的第一个数 = 2×1 + 0×1 = 2
结果的第二个数 = 0×1 + 3×1 = 3
```
结果是 [2, 3]。

**几何含义**：这个矩阵把空间横向拉伸了 2 倍，纵向拉伸了 3 倍。
原来的 [1,1] 箭头，变成了 [2,3] 箭头。

**ML 里的用法**：
- 神经网络的每一层 = 一个矩阵变换（加上激活函数）
- Transformer 里的 Q/K/V 矩阵 = 把输入向量投影到不同的"角度"

---

## 矩阵乘法：变换的叠加

**直觉**：两个矩阵相乘，就是"先做 A 变换，再做 B 变换"。
顺序很重要——先转再拉，和先拉再转，结果不一样。

**例子**：

矩阵 A = [[1, 0], [0, -1]]（上下翻转）
矩阵 B = [[2, 0], [0, 2]]（等比放大 2 倍）

B × A 的含义：先把空间上下翻转，再整体放大 2 倍。

**ML 里的用法**：多层神经网络 = 多个矩阵变换连续叠加

---

## 特征向量与特征值：变换后方向不变的箭头

**直觉**：大多数箭头经过矩阵变换后，方向会改变。
但有一些特殊的箭头，变换后**方向不变**，只是长度变了——这就是特征向量。
长度变成原来的几倍，就是对应的特征值。

**例子**：

矩阵 M = [[2, 0], [0, 3]]（横向×2，纵向×3）

- 箭头 [1, 0]（纯横向）→ 变换后是 [2, 0]，方向没变，长度×2。
  特征向量 = [1, 0]，特征值 = 2
- 箭头 [0, 1]（纯纵向）→ 变换后是 [0, 3]，方向没变，长度×3。
  特征向量 = [0, 1]，特征值 = 3
- 箭头 [1, 1]（斜向）→ 变换后是 [2, 3]，方向改变了，不是特征向量

**ML 里的用法**：PCA 降维 = 找数据分布里"变化最大的方向"，就是找协方差矩阵的特征向量

---

## 行列式：变换后面积的缩放比例

**直觉**：矩阵变换会把空间里的面积（2D）或体积（3D）按某个倍数缩放。
这个缩放倍数就是行列式。

- 行列式 = 2：变换后面积变成原来的 2 倍
- 行列式 = 0：空间被"压扁"了（所有点挤到一条线上），信息丢失
- 行列式 < 0：空间被翻转了（左手系变右手系）

**例子**：矩阵 [[2, 0], [0, 3]] 的行列式 = 2×3 - 0×0 = 6。
原来边长为 1 的正方形，变换后面积变成 6。

**ML 里的用法**：某些概率模型（如流模型）需要计算行列式来追踪变量变换后的概率密度

---

## 遇到线代问题时的处理原则

1. 先问"这个操作在几何上是什么意思"
2. 用 2×2 的最小例子手算验证直觉
3. 再建立它和 ML 场景的联系
4. 最后才考虑数学符号的写法
