# PDF 内容处理流程

本文件定义所有涉及 PDF 的处理逻辑。
触发条件：用户给出页码、提及 PDF 路径、或要求提取课本内容时。

---

## PDF 路径记忆

- 用户**第一次**提供 PDF 路径 → 写入 profile 的「PDF路径」字段，整个对话期间不再询问
- 用户**只说页码**（如"帮我看第 30 页"）→ 直接用 profile 中缓存的路径
- profile 中路径为空 → 询问："你的 PDF 在哪里？"
- 用户明确给出新路径 → 更新 profile 中的路径

---

## 页码偏移

PDF 通常有封面、版权页等占据页码，导致 PDF 阅读器显示的页码与程序内页码不一致。

**偏移量确定流程（仅在 profile 中「页码偏移」为空时执行）：**

1. 提取用户说的第 1 页（程序输入 1）
2. 展示提取到的内容，询问用户："这是你 PDF 里第 1 页的内容吗？"
3. 若**不是**，询问："PDF 阅读器里第 1 页对应程序里的第几页？"根据回答计算偏移量
4. 将偏移量写入 profile：`页码偏移：1`
5. 之后所有页码自动修正：用户说"第 N 页"→ 程序输入 `N + 偏移`

**偏移量已缓存时**：直接使用，不再询问。

## 方式一：用户粘贴文字

用户直接把 PDF 里的段落复制过来。

处理步骤：

1. 读懂原文，提取 1~2 个核心概念
2. 完全忽略原文的表达方式，用大白话重新讲一遍
3. 给出具体数字例子
4. 和学生已知内容建立联系
5. 出一个确认小题

> 注意：李宏毅课程原文假设读者有大学数学基础。
> 任务是把"大学数学语言"翻译成"初中生能懂的语言"，公式推导不要照搬。

---

## 方式二：用户给出页码范围

### 第一步：页码范围校验

```
页码范围 ≤ 10 页  → 直接提取
页码范围 11~20 页 → 先警告，询问是否继续：
  "这个范围有点大（X 页），提取可能会比较慢。
   建议把最想理解的段落复制过来，效果会更好。要继续尝试吗？"
页码范围 > 20 页  → 拒绝：
  "这个页码范围太大了（X 页），处理起来很可能超时。
   推荐做法：在 PDF 里找到你最不懂的那段，把那段文字复制过来，我来帮你解释。"
```

### 第二步：确定 Python 解释器

检查 profile 的「环境配置」区块：

- `python解释器` 字段非空且可执行 → **直接使用，跳过后续步骤**
- 字段为空或路径失效 → 读取 `references/env_setup.md`，按其流程执行

### 第三步：定位脚本路径

检查 profile 的「PDF配置」区块：

- `提取命令示例` 字段非空 → **直接替换其中的页码参数后运行**，跳过路径定位
- 字段为空 → 从 profile 的「基础配置」读取 `Skill目录` 绝对路径，拼接脚本路径：

```
脚本路径 = <Skill目录>/scripts/extract_pdf_pages.py

# 示例（Windows）：
D:\WorkSpace\AISec\.claude\skills\ml-tutor\scripts\extract_pdf_pages.py

# 示例（Linux/macOS）：
/home/user/project/.claude/skills/ml-tutor/scripts/extract_pdf_pages.py
```

> `Skill目录` 在初始化时已写入 profile，直接读取即可，不需要搜索文件系统。

### 第四步：执行提取

从 profile 的「基础配置」读取 `操作系统` 字段，选择对应的执行方式：

**macOS / Linux：**
```bash
"<PYTHON>" "<SCRIPT>" "<pdf路径>" <起始页+偏移> <结束页+偏移>
```

**Windows（Git Bash）：**
```bash
# Claude Code 在 Windows 下使用 bash，直接用 Unix 风格设置环境变量，避免 cmd /c 的嵌套引号问题
PYTHONUTF8=1 "<PYTHON>" "<SCRIPT>" "<pdf路径>" <起始页+偏移> <结束页+偏移>
```

> 平台信息已在初始化时写入 profile，这里直接读取，不需要重新判断。

**首次成功运行后**，将完整命令示例（含实际路径）写入 profile 的「提取命令示例」字段。
下次遇到 PDF 请求时，直接从该字段读取命令，只替换页码参数即可。

### 第五步：处理提取结果

**提取成功，内容正常** → 按「方式一」的步骤处理：用大白话重新讲，给数字例子，出确认小题

**提取成功，但内容标注了乱码警告** → 告知用户：
> 这几页里有公式图片，自动提取的文字可能不完整。
> 建议你在 PDF 阅读器里找到那段，手动复制文字给我，这样我能完整帮你解释。

**提取完全失败** →
> 请把你想理解的段落直接复制过来，我帮你解释。
