# 审计报告模板

输出审计报告时必须包含以下 5 个区块，按顺序输出。不要遗漏任何部分。

---

## 区块一：基本信息

表格，6 个字段：Skill 名称、SKILL.md 行数、References 文件（数量+总行数）、Scripts 文件（数量）、目录总大小、核心逻辑概括（一句话）。

## 区块二：维度评分总览

7 行表格，每行包含：维度编号、维度名称、评分（X/5 或 N/A）、一句话总结。

表格下方给出**综合评分**：各维度等权平均，N/A 不计入分母。

## 区块三：逐维度详细分析

每个维度独立一节，每节包含：

- **核心逻辑/文件结构/矛盾点/信息密度/脚本环境/文件场景/规范检查 分析**（根据维度选择对应的分析角度）
- **关键发现**：逐条列出，每条附原文引用（文件 + 行号或原文摘录）
- **改进建议**：逐条给出，具体可操作（"把 X 拆成 Y"而非"这里可以更好"）

D4 和 D5 可能为 N/A，此时只需说明"不适用"及原因，无需展开分析。

## 区块四：优先改进清单

表格，按优先级（高→中→低）排列，列：优先级图标、维度编号、问题简述、建议简述。

## 区块五：总体评价

一段话总结 skill 的优缺点，以及是否推荐使用。

---

## 格式约定

- 报告使用中文
- 评分维度名称保留英文简写（D0, D1, ...）
- 顶部标注：`# Skill 审计报告：<skill-name>`，附带审计时间和 skill 路径
- 每个改进建议以 `- ` 开头，必须包含具体动作（动词开头）
